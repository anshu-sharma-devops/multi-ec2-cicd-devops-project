☸ Kubernetes Beginner Guide

A simple visual guide created as part of the Multi-EC2 CI/CD DevOps Project. This page explains Kubernetes from scratch using the same project workflow: Docker → ECR → Kubernetes → Service → Website.

🚀 Why Kubernetes?

When applications run inside containers, managing one container is easy. But in real projects, we need multiple containers, automatic restart, scaling, load balancing, and zero-downtime deployment. Kubernetes solves these problems by managing containers automatically.

🧠 Core Concepts

📦 Container

A container packages the application with everything it needs to run.

☸ Pod

The smallest unit in Kubernetes. A pod runs one or more containers.

📊 Deployment

Manages replicas of pods and keeps the application running.

🌐 Service

Exposes pods using a stable network endpoint.

🧩 ReplicaSet

Ensures the desired number of pod replicas are always running.

🖥 Node

A machine where Kubernetes runs application workloads.

🏗 Project Architecture

GitHub → Jenkins → Docker Build → Amazon ECR → Kubernetes Deployment → Service → Website

📄 Kubernetes Deployment YAML

apiVersion: apps/v1
kind: Deployment
metadata:
  name: devops-web
spec:
  replicas: 2
  selector:
    matchLabels:
      app: devops-web
  template:
    metadata:
      labels:
        app: devops-web
    spec:
      containers:
      - name: devops-web
        image: devops-web:v3
        imagePullPolicy: Never
        ports:
        - containerPort: 80

🌐 Kubernetes Service YAML

apiVersion: v1
kind: Service
metadata:
  name: devops-web-service
spec:
  selector:
    app: devops-web
  ports:
  - port: 80
    targetPort: 80
  type: NodePort

🧪 Verification Commands

kubectl get nodes
kubectl get pods
kubectl get deployment
kubectl get svc
minikube service devops-web-service --url
curl $(minikube service devops-web-service --url)

🛠 Problems I Faced & Fixed

❌ ImagePullBackOff

Kubernetes could not find the local Docker image. I fixed it by building the image inside Minikube's Docker environment and setting imagePullPolicy: Never.

❌ No Space Left on Device

Docker and Minikube filled the root disk. I increased the EBS volume to 20 GB and resized the Linux filesystem.

❌ Low Memory

t3.micro was not enough for Kubernetes. I upgraded the App Server to a larger instance for Minikube practice.

🎯 Final Result

The Dockerized DevOps website was successfully deployed inside Kubernetes using Minikube. The application runs through Kubernetes Deployment and is exposed using a NodePort Service.
